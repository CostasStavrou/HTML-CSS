User-oriented computing

Of course this is not original research. These ideas exist many years and many of them have been tried and failed on the field.
Please hover on top of the following items.

The past and the present

In the history of computing many ideas where presented related to the interface between the user and the computer system. Many companies tried to invent the next big think in User Interface, to be able to control the next wave of computing and make a lot of money. Most of the UIs where application-centric with the support of the notion of files and directories. The user had to learn applications and adopt into a world of hierarchies. A world that some thought was too difficult to grasp for mere humans. Just to remember some of the attempts against these ideas we can look

  • the document-centric model of OpenDoc applications
  • task-oriented user interface with the help of guides in many Microsoft products
  • nowadays the idea that users should not save locally, but only on the cloud.

Especially the idea of the cloud is the most limiting UI for the user. This is enforced in some occasions to the user by eliminating for example the idea of external storage (SDCards slots absent on phones/tablets).

The Problem

Mainly the enforcement part. The point is that these UI don't support the many styles of interfacing, but only one style by making assumptions for the user. A known one is the difficulty we have to organize data in files and folders, an idea that most humans are capable to grasp since they use folders to organize information outside of the virtual world of computing since forever (ie. long before computers existed).

This way the user is not capable to access, archive and exchange files without the use of the cloud. And the cloud is not always available. This way the user has no control of it's data. The control of data is passed to the companies we trust to use on the Internet.

A proposal

These days technology has advanced tyo the point that the computing power we have on a small device like a smartphone can support things that in the past where to demanding to being thought as a solution.

So a combination of a revision control system like Git with the ideas of HyperText (link one piece of data to another), compression algorithms and ability to mount a compressed file on a file system can be the solution we need. Simple, based on already proven technologies and at the same time simple and transparent to the user. A file with a folder containing all the necessary assets can be compressed and mounted whenever accessed.
